# ***Flyweight*** 享元模式
>运用共享技术有效支持大量细粒度的对象
## 适用性：
* 一个应用程序使用了大量的对象
* 由于使用大量对象，造成很大的储存开销
* 对象的大多数状态可以变为外部状态；
* 如果删除对象的外部状态，可以用较少的共享对象代替大量对象
* 应用程序不依赖对象标示
## 实现：
* 删除外部状态，理想的状态是外表状态可以由一个单独的对象结构计算得到，且该结构的储存要求非常小
* 管理共享对象。可以使用关联储存查找需要的对象，可以使用引用计数来回收不需要的对象；

## 感受：
* 享元工厂有点类似单例模式的单例注册表，并且享元工厂可用工厂方法模式实现；
